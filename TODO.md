# 待优化项

按"价值 / 成本"分级整理。推荐从 🔥 区按顺序做，投入小、体感大。

---

## 🔥 高价值·低成本

### 1. 快速重种
**场景**：收获后手动删记录 → 重开"新种作物"表单 → 选同类型 → 填时间。
**方案**：已收获卡片加"↻ 重种"按钮，点击后：
- 自动用同 type、`planted_at = now`、是否立刻浇水沿用上次
- 原记录直接标删或保留 30min 自动清理兼容并存
**预估**：10 分钟。改 `cropCard` 模板 + 调 `addCrop` 复用逻辑。

### 2. 顶部汇总栏
**场景**：作物多了，看不清"还有几个要浇、几个能收"。
**方案**：头部横栏显示：
- 🌱 进行中 N · 🔥 待浇 M · 🌾 可收 K · 💧 下次浇水还有 HH:MM
**预估**：15 分钟。新增 computed property 聚合 `crops.map(cropState)` 结果。

### 3. 按紧迫度排序
**场景**：当前按种植时间排，紧迫的作物可能排到列表底。
**方案**：默认排序键 = `shouldWaterNow ? 0 : windowA`，可收获的排最前，再按 windowA 升序，已收获的沉底。
**预估**：5 分钟。改 `sortedCrops` getter。

## 💡 中等价值

### 4. 深色模式
**方案**：Tailwind 的 `dark:` 前缀 + `prefers-color-scheme` 自动切，或顶部加 🌙 开关写 localStorage。
**预估**：30 分钟。需要遍历现有类名补 `dark:` 变体。

### 5. 撤销误操作
**场景**：手抖点了"浇水"或"收获"，目前只能重新插记录。
**方案**：浇水/收获后右下角弹 Toast："已浇水 · 撤销"，10s 内可撤。内部调 `update` RPC 回滚 water_events / harvested_at。
**预估**：45 分钟。需新增 RPC 的 `undo_water` / `undo_harvest` action。

### 6. 输入记忆
**方案**：上次选过的 type / 是否立刻浇水存 localStorage，下次打开表单自动预填。
**预估**：10 分钟。

## 🎨 视觉类

### 7. 作物图标多样化
**方案**：新种作物时让用户挑个 emoji（🌾🌽🥕🍓🍅🌶️🍆），或按名字首字母 hash 映射。
**预估**：20 分钟。schema 已有 name 字段，复用即可。

### 8. 进度条动画
**场景**：湿润度条从 100% 跳到 67% 是硬切。
**方案**：Tailwind `transition-all duration-1000` 加一行，条变化有补间动画。
**预估**：5 分钟。

## ⏰ 功能类

### 9. 浇水历史时间轴
**方案**：卡片底部可展开，竖直时间轴展示所有 water_events + 种植/收获时间点。
**预估**：30 分钟。纯前端，数据已在 water_events。

### 10. 成熟提醒音效
**场景**：进入"最后一次浇水"或"立刻收获"时播短音效。
**方案**：Web Audio API 播 0.3s beep，用户首次交互后才允许（浏览器限制）。可加开关。
**预估**：20 分钟。iOS Safari 需用户主动开声音。
